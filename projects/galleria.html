이전 페이지로

갤러리아 전자영수증

한화 갤러리아 백화점 앱에서 사용되는 전자영수증 및 백오피스를 개발하는 프로젝트이며, 개발단계는 사내에서 진행하였고 배포 및 테스트 단계는 한화 본사에 며칠간 상주하면서 진행하였습니다.

참여인원 및 업무
  • 팀장 - 업무 협의 및 기획
  • 안드로이드 개발자 1명 - 갤러리아 앱 설정화면 및 전자영수증 프레임 개발
  • 아이폰 개발자 1명 - 갤러리아 앱 설정화면 및 전자영수증 프레임 개발
  • 영수증데이터 수집 개발 1명 - 갤러리아 백화점 각 지점의 영수증 정보 수집 프로세스 개발
  • 웹개발자 1명 - 갤러리아 앱 전자영수증 화면 개발, 백오피스 개발
  • 퍼블리셔 1명 - 웹 퍼블리싱(HTML, CSS, IMAGE)
My 상세 업무
  • 갤러리아 전자영수증 리스트 화면 개발
  • 갤러리아 전자영수증 상세 화면(앱, 문자 발신용 2가지 버전), 바코드, PDF 다운로드 기능 개발
  • 앱 설정 - 전자영수증 ON/OFF 업데이트 API 제공
  • 갤러리아 회원정보조회 API 연동(APP Token 조회)
  • 전자영수증 백오피스 개발
  • 한화 통합 업무 시스템(Circle)과의 백오피스 시스템 로그인 연동(SLO)
  • Jeus & WebtoB 배포 및 한화 서버 담당자와의 협업(Jeus & WebtoB, 앱토큰, SLO 서버 관련 방화벽 문제 등)

애플리케이션 구성

  • Oracle Java 1.8
  • Springframework 4.3.25.RELEASE
  • JNDI를 사용하여 Jeus에 등록된 DataSource(Oracle) 사용
  • Mybatis
  • Maven
  • JSP, JSTL, javascript, Jquery, barbecue, itextpdf, swiperjs
  • SVN

서버 구성 및 배포

IBM AIX 웹 서버(WebtoB) 2대, WAS 서버(JEUS, Java8) 2대, 운영 데이터베이스(Oracle) 서버, NAS 서버, 테스트 WAS, 데이터베이스(Oracle) 서버로 구성되어 있으며,
배포는 내부망에서 SFTP로 웹 또는 WAS 서버에 파일을 업데이트한 뒤 webadmin에서 jeus를 재기동 하였습니다.

인터페이스

갤러리아 앱, 전자영수증 웹페이지, 토큰인증API(한화) 간의 인터페이스를 통해 회원의 영수증 정보를 가져오도록 개발 되었습니다.
회원이 앱에 로그인하여 전자영수증 페이지를 요청할 때 앱 토큰 값을 파라메터도 함께 전송합니다.
웹페이지에서 수신한 앱 토큰 값을 토큰 인증 API로 전송하여 회원ID를 Response 받은 뒤 데이터베이스에 조회하여 회원 영수증 리스트를 화면에 출력합니다.



토큰인증API는 한화 측 시스템으로 Rest API로 제공되었으며 String 토큰 값에 대해 JSON 형식으로 응답을 주었습니다.



한화 통합 업무 시스템(Circle)과의 로그인 연동(SLO)을 통해 관리자 백오피스에 로그인 하는 인터페이스도 포함되어 있습니다.



백오피스

갤러리아 전자영수증 발급 조회 및 통계, 마케팅 관리 등을 관리자가 이용할 수 있도록 웹으로 개발된 백오피스입니다.
